# Security Policy

## Scope

This repository primarily contains technical documentation, laboratory methodologies, analytical examples, scripts, and portfolio projects.

---

## Reporting Security Issues

If you discover:

- exposed credentials
- sensitive information
- confidential data
- licensing concerns
- security vulnerabilities within repository scripts

please do **not** open a public issue.

Instead, contact the repository owner directly using the professional contact information provided in the README.

---

## Responsible Disclosure

Verified issues will be investigated promptly.

If required, affected content will be corrected, removed, or updated in the next repository release.

---

## Data Protection

No confidential laboratory data, patient information, proprietary industrial information, or restricted research materials should ever be committed to this public repository.

All published datasets should be:

- Publicly available,
- Properly anonymized,
- Synthetic, or
- Approved for public release.
